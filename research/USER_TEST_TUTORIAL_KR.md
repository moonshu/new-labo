# 사용자 테스트 튜토리얼 (쉬운 용어 버전)

이 문서는 기능을 처음부터 끝까지 직접 써보면서 문제를 찾기 위한 안내서입니다.

## 0. 용어 먼저 이해하기

1. 질문(DRAFT): 아직 결론이 없는 생각 메모
2. 유보(HESITATED): 결론을 내리기 전 임시 상태
3. 정리(CONCLUDED): 근거를 붙여 저장한 결론
4. 근거(Evidence): 링크/노트/문서 같은 판단 이유
5. 관계성(Graph): 생각끼리 연결/반박된 구조

## 1. 시작하기

1. 브라우저에서 `http://localhost:3000` 열기
2. 왼쪽 사이드바에서 문제(질문) 하나 선택
3. 가운데 입력창에 생각을 1~2문장 입력
4. `분석` 버튼을 눌러 제안 확인

## 2. 가장 쉬운 기본 플로우 (필수)

1. 질문 상태 저장
- 입력: 결론이 없는 문장
- 기대결과: `질문` 또는 `유보`로 저장 가능

2. 결론 상태 저장
- 입력: 결론형 문장
- 행동: 근거 1개 추가 후 저장
- 기대결과: 저장 성공

3. 결론 상태에서 근거 없이 저장 시도
- 기대결과: 저장 차단 + 안내 문구 표시

## 3. 기능별 점검 순서 (로드맵)

1. 검색/필터
- 상단 검색창으로 키워드 검색
- 상태 필터를 `근거 경고`, `AI 생성`으로 바꿔보기

2. 타임라인
- 노드 클릭 → 우측 패널 열림 확인
- 날짜/상태/근거 경고 배지 확인

3. 우측 패널(관계성/수정)
- 노드 내용 수정 후 저장
- 상태를 `정리`로 바꾸고 근거 없을 때 경고 확인
- `유보 상태로 전환` 버튼 동작 확인

4. 사이드바
- 문제 추가/이름변경/삭제 동작 확인
- `데이터 도구 열기` 후 내보내기/가져오기/초기화 버튼 확인

5. 보기 설정
- `지표 보기` 버튼으로 상단 지표 패널 토글
- `용어 쉬운 설명` 버튼으로 설명 패널 토글
- 라이트/다크 전환 버튼 확인

## 4. 발견한 문제 기록 방식

문제 하나당 아래 4가지만 적어주세요.

1. 어디서 발생했는지: 예) 타임라인 > 필터 > AI 생성
2. 내가 한 행동: 클릭/입력 순서
3. 기대한 결과
4. 실제 결과 (가능하면 스크린샷 포함)

## 5. 빠른 회귀 테스트 (3분)

1. 새 노드 추가
2. 검색으로 찾기
3. 우측 패널에서 수정 후 저장
4. 새로고침 후 데이터 유지 확인
