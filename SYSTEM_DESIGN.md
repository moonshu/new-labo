# 🌌 사유 시스템 (Thought Mapping System) 상세 기획 및 UI/UX 설계서

본 문서는 `AI_HANDOFF.md`의 철학과 `INFRA_PLAN.md`의 기술 스택을 바탕으로, 실제 구현될 웹 애플리케이션의 **구체적인 화면 레이아웃, 사용자 흐름(User Flow), 컴포넌트 단위의 인터랙션 명세**를 정의합니다. 개발 돌입 시 프론트엔드 파트의 바이블로 활용됩니다.

---

## 1. 전역 레이아웃 및 UI 토폴로지 (Global Layout & Topology)

시스템은 '하나의 빈 캔버스'라는 철학을 유지하기 위해, 복잡한 네비게이션을 숨기고 핵심 기능만 노출하는 **Single Page Application (SPA)** 형태의 뷰 전환을 사용합니다.

### 🍅 화면 분할 (Grid Layout)
- **Left Sidebar (주제/문제 인덱스)**
  - 평소에는 축소(Collapsed)되어 있거나 반투명하게 존재하여 시선을 빼앗지 않습니다.
  - 마우스를 올리거나(Hover) 햄버거 메뉴 클릭 시 확장됩니다.
  - 상단: `연속 기록(Streak) 잔디 위젯`과 `진행 중인 문제 갯수 요약`
  - 중단: `Themes (Project/Routine)` 목록 (사용자 지정)
  - 하단: `Problems (열려있는 질문들)` 목록
- **Center Canvas (메인 작업 공간)**
  - 화면의 70% 이상을 차지하는 거대한 여백. 기본적으로 항상 `The Canvas(에디터)`가 위치합니다.
  - 특정 문제나 테마를 클릭 시 에디터 상단에 `The Timeline`이 전개됩니다.
- **Right Drawer / Popover (The Graph 및 메타데이터)**
  - 평소에는 숨겨져 있으며, 특정 사고 노드(Thought Node)를 선택하거나 AI의 분석이 완료되었을 때 우측에서 부드럽게 등장합니다.
  - 연결된 다른 생각(Backlinks)이나 AI가 제안하는 [관계성]이 위젯 형태로 표시됩니다.

## 2. 핵심 컴포넌트별 상세 인터랙션 (Component Specs)

### 2.1. `The Canvas` (사고력 입력기)
가장 많이 마주하게 될 입력 창입니다.
- **상태 변화**:
  - `Idle`: 화면 중앙에 placeholder("지금 맴도는 생각은 무엇인가요?") 하나만 있는 텍스트 에어리어.
  - `Typing`: 포커스가 가면 주변 UI(사이드바 등)의 opacity가 30%로 낮아져 글쓰기에만 몰입하는 **Focus Mode** 발동.
  - `Pending Analysis`: 글을 마치고 `Cmd+Enter` (또는 저장 버튼)를 누르면 에디터가 읽기 전용으로 변하고 은은한 로딩 스피너(별무리 애니메이션) 표시.
  - `Suggestion Ready`: 분석 완료 시 입력창 바로 아래에 AI의 제안 칩(Chips)들이 스르륵 나타남.
- **인터랙션(AI 제안 수락/거절)**:
  - 제안된 칩 (예: `문제: 개인의 한계?`, `주제: AI 시스템 설계`, `판단: 유보`)을 클릭하면 확정(파란색으로 변경).
  - 칩 우측의 `X`를 누르거나 무시하고 새 글을 쓰면 제안 폐기.
- **💥 의도적 파편화 기능 (`[?]` 버튼)**:
  - 쓰다가 막혔을 때 에디터 우측 하단의 `[?] 아직 모르겠음` 아이콘을 클릭.
  - 즉시 해당 글은 "유보 상태의 파편"으로 저장되며, 에디터는 다시 깨끗하게 비워짐.

### 2.2. `The Timeline` (시간 차원 뷰어)
선택한 문제(Problem)에 대한 사유의 궤적을 보여줍니다.
- **구조**: 최하단이 가장 과거의 "최초 질문(First Node)", 위로 올라갈수록 가장 최근의 생각으로 렌더링.
- **노드 디자인 (Thought Card)**:
  - 카드 형태가 아닌, 여백이 많은 타이포그래피 중심의 스레드(Thread) 형태.
  - 카드 우측 상단에 흐릿하게 `체류 시간(ex: +12일 뒤 추가된 생각)` 표시.
- **스레드 잇기**: 타임라인 맨 하단(또는 맨 상단)에 위치한 캔버스 창에서 글을 쓰면, 현재 문제에 자동으로 귀속되어 스레드가 이어집니다.

## 3. 게미피케이션 & 강제 회고 장치 상세 시나리오

외적 보상이 아닌, 나 자신의 사고를 책임지게 만드는 장치들의 동작 원리입니다.

### 3.1. `Reflection Lock` (인지적 병목 / 강제 회고 잠금)
- **발동 조건 계산 (Server-side)**:
  - 매번 앱 로드 시 상태 스토어(`useSystemStore`)에서 미해결 문제 리스트 조회.
  - `status == 'OPEN'` 이고 `now() - last_thought_at > 72 hours (3일)`인 문제가 1개 이상 존재할 경우 Lock 발동.
- **UI 경험 (Overlay)**:
  - 앱에 접속하자마자 화면 전체가 블러(Blur) 처리되고 팝업 오버레이가 화면을 덮습니다.
  - 메시지: *"3일간 방치된 질문이 있습니다. 사고를 이어가거나 명시적으로 보류/종결해야 새로운 생각을 넣을 수 있습니다."*
  - 오버레이 중앙에는 방치된 문제의 가장 마지막 [사고 노드 원문]이 강제 노출됩니다.
- **해제(Unlock) 방법**:
  - 해당 노드 아래에 단 한 글자라도 덧붙이거나(새로운 노드 발행).
  - 문제의 상태 속성에서 명시적으로 `[보류하기(Pending)]` 버튼을 눌러 휴면 상태로 전환.

### 3.2. `Contextual Nudge` (맥락적 넛지 칩)
- **발동 시점**: 사용자가 캔버스를 빈 상태로 1분 이상 응시하거나, 3일 이상 접속이 뜸했다가 들어왔을 때 우측 상단이나 빈 캔버스 하단에 토스트(Toast) 메시지처럼 나타남.
- **패턴**:
  - *충돌 넛지*: "이전에 [A]라고 하셨는데, 지금 [B]의 맥락과 연결될 수 있을까요?"
  - *먼지떨이 넛지*: "100일 전에 기록한 [이 문제]를 아직도 품고 계시네요."
- **동작**: 클릭 시 해당 문제의 타임라인으로 바로 이동.

### 3.3. `Thought Streak` (연속 기록 시각화)
- 사이드바나 프로필 영역에 위치하는 Git 잔디 플러그인과 유사한 7x5 (또는 7x10) 그리드.
- 각 셀의 진하기는 하루 동안 생성한 '사고 노드의 갯수' 또는 '사용한 단어 수'에 비례.
